架构与部署

mfm-h5 是面向多业务 H5 页面的独立前端工程，当前首批落地「房产榜单」模块。

三端分工

mfm-uniapp-vue3（小程序） web-view 壳页 → VITE_MFM_H5_ORIGIN/v2/pages/<deployPath> mfm-h5（本仓库） Vue SPA · base /v2/ · history 路由 mfm-api REST · 如 GET /data/sell-rank

与房小团 FXT 的关系

原站 H5 为 uni-app 编译的静态包（Vue chunk + uni.webview）。本仓库选用 Vite + Vue 3 以降低维护成本，但对齐公开 URL 与小程序桥接行为。

构建与 Nginx

  • pnpm build → dist/
  • vite.config.ts 中 base: '/v2/'
  • 生产示例：https://h5.example.com/v2/pages/data/sell-rank
  • 微信「业务域名」需配置 H5 根域名

环境变量

变量用途
VITE_DEV_PORT本地 dev 端口，默认 5180
VITE_API_BASE_URLFastAPI 根；dev 经 proxy /api
VITE_DEFAULT_CITY_ID默认城市 ID（如 370100）

小程序侧使用 VITE_MFM_H5_ORIGIN（在 mfm-uniapp-vue3 配置）。
