架构与部署

mfm-h5 是面向多业务 H5 页面的独立前端工程，当前首批落地「房产榜单」模块。

两层边界（为何用 web-view）

榜单等页走独立 H5，是为了 改页面/站内导航不必发小程序版。 小程序只负责用配置拼出 web-view 的 src；H5 内再用 Vue Router 跳转即可。

层仓库真值来源
壳mfm-uniapp-vue3GET /config/page → props.webviewSrc（page-key）
页mfm-h5rank-routes.ts 注册路径；站内 router.push；业务 GET /data/…
小程序壳 page-key=data-sell-rank → page_config.webviewSrc = {H5}/v2/pages/data/sell-rank → mfm-h5 SellRank.vue → GET /data/sell-rank 榜单中心在 H5 内 router.push 子榜路径 → 仅部署 mfm-h5 即可调整

与房小团 FXT 的关系

原站 H5 为 uni-app 编译的静态包（Vue chunk + uni.webview）。本仓库选用 Vite + Vue 3 以降低维护成本，但对齐公开 URL 与小程序桥接行为。

构建与 Nginx

  • pnpm build → dist/
  • vite.config.ts 中 base: '/v2/'
  • 生产示例：https://h5.example.com/v2/pages/data/sell-rank
  • 微信「业务域名」需配置 H5 根域名

环境变量

变量用途
VITE_DEV_PORT本地 dev 端口，默认 5180
VITE_API_BASE_URLFastAPI 根；dev 经 proxy /api
VITE_DEFAULT_CITY_ID默认城市 ID（如 370100）

小程序侧使用 VITE_MFM_H5_ORIGIN（在 mfm-uniapp-vue3 配置）。
